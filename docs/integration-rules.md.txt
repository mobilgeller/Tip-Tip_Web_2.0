# Integration Rules

## Architecture

The current website is hosted through GitHub Pages.

Future integrations must not assume any specific platform unless explicitly decided.

## Possible Future Platforms

The project may later integrate with:

- Wix
- Custom backend
- Supabase
- Firebase
- Booking platforms
- CRM systems
- Marketing automation platforms
- Custom APIs

The choice of platform is not finalized.

## Core Principle

The website must remain platform-independent whenever possible.

Avoid coupling the core website structure to a single provider.

## Integrations

New integrations should be:

- modular
- replaceable
- loosely coupled
- easy to remove

## Booking Systems

Booking functionality may be implemented through:

- Wix
- external booking software
- custom booking system
- CRM integration

No specific solution is currently mandated.

## Content Management

Blog, knowledge base, promotions and seasonal offers may later be managed through:

- Wix
- headless CMS
- custom admin panel
- other CMS solutions

The implementation choice remains open.

## Important

Do not redesign the website around any specific platform unless explicitly requested.

Preserve flexibility for future architectural decisions.
